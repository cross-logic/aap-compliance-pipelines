|===

All job templates use the following common settings:

* *Project*: `compliance-content`
* *Inventory*: `compliance-rhel9-inventory` (default; overridable at launch)
* *Organization*: `compliance-prototype`
* *Prompt on Launch: Limit*: Enabled (allows scoping to specific hosts)

== API endpoints

The compliance pipelines Backstage plugin communicates with {ControllerName} through the platform gateway. All endpoints use the `/api/controller/v2/` base path.

[cols="2,1,3", options="header"]
|===
|Endpoint
|Method
|Purpose

|`/api/controller/v2/workflow_job_templates/`
|GET
|List workflow job templates available for compliance scans

|`/api/controller/v2/workflow_job_templates/<id>/launch/`
|POST
|Launch a compliance scan workflow with extra variables, limit, and job tags

|`/api/controller/v2/workflow_jobs/<id>/`
|GET
|Check the status of a running or completed workflow job

|`/api/controller/v2/workflow_jobs/<id>/workflow_nodes/`
|GET
|Retrieve the status of individual nodes in a workflow run

|`/api/controller/v2/jobs/<id>/`
|GET
|Check the status of an individual job within a workflow

|`/api/controller/v2/jobs/<id>/job_events/`
|GET
|Retrieve job events including scan findings parsed from runner output

|`/api/controller/v2/jobs/<id>/stdout/`
|GET
|Retrieve raw job output for log display

|`/api/controller/v2/inventories/`
|GET
|List inventories available for scan targeting

|`/api/controller/v2/execution_environments/`
|GET
|List execution environments for compliance profile registration
|===
